## 1. Хэрэв шалгалт өнөөдөр болбол би энэ кодыг өөрөө бичиж чадах уу?

**Хариулт: Хэсэгчлэн (~60-70%)**

### Чадах хэсэг

- Express boilerplate ойлгож тайлбарлаж чадна (app.js, server.js, middleware дараалал)
- Layered architecture (Routes Controller Service Repository) ба Separation of Concerns зарчмыг тайлбарлаж чадна
- HTTP status codes ялгаа мэднэ (201 create, 200 read, 204 delete, 400 validation, 404 not found, 500 server)
- Prepared statements ба SQL injection-ийн тухай тайлбарлаж чадна
- Error middleware-ийн 4 параметр (err, req, res, next) яагаад special signature гэдгийг мэднэ
- ES modules ба CommonJS-ийн ялгаа (import vs require)

### Эргэлзээтэй хэсэг

- Zod schema-ийн нарийн API (refine, transform, coerce) - жишээгүйгээр санахад хүндрэлтэй
- Mermaid sequenceDiagram-ийн alt блок syntax - санахад эргэлзээтэй
- jest.unstable_mockModule pattern - ES modules-той Jest ашиглах нарийн арга
- OpenAPI 3.0-ийн нарийн ($ref, oneOf, allOf) - жишээгүй санахад хэцүү

### Чадахгүй хэсэг

- Regular expression (regex) - AI-аар бичүүлдэг болсон, өөрөө бичих дасгал хэрэгтэй
- better-sqlite3-ийн нарийн API (transactions, prepared statement cache strategy)

## 2. Дахин хийнэ гэвэл юуг өөрөөр хийх вэ?

### Гол өөрчлөлт

1. **Шахмал хуваарьт хүрэхгүй байх.** 14 хоногтой бие даалтыг 11 хоногт хийсэн нь stress нэмсэн. Эхний өдөр дотор төлөвлөгөө тогтоож, өдөр бүр маргаашид commit хийдэг dadalтай болох.

2. **AI session log-ыг build хийх явцад тэмдэглэх.** Day 6-нд бүх Report-ыг гэнэт бичих гэж оролдох нь муу санаа. Өдөр бүр 5 минут зарцуулж тэмдэглэх dadal сурах.

3. **Кодоо тайлбарлах дасгал илүү хийх.** AI үүсгэсэн файл бүрийг "найз руу explain" хэлбэрээр уншиж тайлбарлах. Шалгалтад бэлэн байх хамгийн зөв арга.

4. **Encoding asuudal-аас эхнээс зайлсхийх.** Notepad биш, VS Code-ийг суулгаж эхэлсэн бол 2 удаагийн encoding asuudал гарахгүй байсан.

5. **Test-ийг feature-аас өмнө бичих (TDD).** Энэ удаа feature implement test гэж явсан. Хэрэв test-ыг урьдчилан бичсэн бол AI-ийн санал болгосон код илүү тодорхой шаардлагатай нийцэх байсан.

### Жижиг өөрчлөлт

- TypeScript-ийг шууд эхнээс ашиглах (zod schema-аас type infer хийх)
- `.editorconfig` файл нэмэх (consistency)
- husky + lint-staged pre-commit hook ашиглах
- GitHub Actions CI ажиллуулж pull request бүрд test шалгах
- Environment variables-ийг `.env`-аас унших (одоогийн hardcoded values)

## 3. Энэ туршлагаас юу сурсан бэ?

### Техникийн талаас

1. **AI бол multiplier, бэлтгэл биш.** Боловсорсон ур чадвартай бол AI-аар хурдан ажиллана. Бэлтгэлгүйгээр AI байсан ч "ажиллах" код гарна, гэхдээ ойлгохгүй.

2. **CLAUDE.md ба slash commands бол project-ын файл, хувийн биш.** Эдгээрийг repo-д commit хийсэн нь өөр оюутан / ирээдүйн би өөрөө ашиглахад тус болно.

3. **Layered architecture бол overkill биш.** Эхэндээ "жижиг проектод 4 layer хэт олон" гэж бодсон ч test-ыг service-аас mock repository-аар тестлэх боломж нь утгатай байсан.

4. **OpenAPI spec-ийг гар аргаар бичих нь сайн дасгал.** Үүнээс хойш REST API дизайн хийхэд route-ы талаар тодорхой бодох болсон.

### Process талаас

1. **"Verify, don't trust"** - лекц 13-аас сурсан энэ зарчмыг практикт хэрэглэхэд яагаад чухал болохыг ойлгов. Express 4 vs 5 hallucination нь нэг өдрийг идсэн. AI хариунд скептик хандаж асуудал тэмдэглэх дадал.

2. **Conventional Commits** нь зөвхөн стандарт биш - git history-ийг товчоор унших боломжтой болгодог. Бие даалтын review хийхэд "юу өөрчлөгдсөн" гэдгийг хурдан харж болно.

3. **AI session log хадгалах нь reflective practice.** Дахин эргэн харахад "AI ийм асуудалтай байсан, бид яаж шийдвэрлэсэн" гэдгийг ярих чадвартай.

### Хувийн талаас

1. **Хариулт хүлээх биш - асуулт сайжруулах.** AI-ийн хариулт нь асуулт чанартай уялдсан. "Энэ зөв үү?" гэхээс "Энэ ямар edge case дээр fail болж болох вэ?" гэж асуухад илүү гүн хариу авдаг.

2. **AI-аар хийсэн кодыг өөрөө тайлбарлах сорилт** нь скилл атрофи-аас хамгаалахын хамгийн сайн арга. Жишээ нь өөрийн кодыг найз руу explain хийх. Тайлбарлахад дутуу мэдрэмж бий бол - мэдэхгүй гэсэн үг.

3. **Документ бичих ур чадвар** - энэ бие даалт нь миний хувьд code бичихээс илүү документ бичсэн. ARCHITECTURE.md, ADR, AI Usage Report - эдгээр нь ирээдүйн ажилд хамгийн чухал ур чадвар.

4. **Үнэнч байх нь чухал.** Шахмал deadline-д "AI-аар бичүүлчих" гэсэн уруу татлага гарч магадгүй. Гэхдээ Report-ыг өөрийн жинхэнэ туршлагатай тааруулах нь шалгалтанд бэлэн байх цорын ганц зам.

### Энэхүү бие даалтын утга

Эхэндээ "AI ашиглах нь хичээл" гэж хачин санагдсан. Эцэст нь ойлгоход энэ бол AI-ийн өмнө код бичиж байсан туршлагаа эргэн харж, AI-ийн дараа хэрхэн зөв ажиллах вэ гэдэгт бэлдэх хичээл.
