**Огноо:** 2026-05-01
**Зорилго:** Бие даалтын сэдэв ба stack-аа сонгох

**Q1:** "Бие даалт 13-д ямар сэдэв сонгох вэ?"

## Session 2 — Scope тогтоох (Day 2)

**Огноо:** 2026-05-02
**Зорилго:** 5 feature хийх үү, эсвэл багасгах уу

### Гол ярилцлага

**Q:** "Tag/Label feature-г хийх үү?"

**AI санал:** Tag нь many-to-many relationship — 2 нэмэлт хүснэгт (tags, task_tags) хэрэгтэй. Time-нэйж эрсдэлтэй. Хэрэв шалгалтад "tag-н JOIN query тайлбарла" гэж асуувал хариулах хүндрэлтэй.

**Шийдвэр:** Tag-г хасч **4 feature** хийе:
1. Task CRUD
2. Due date + Priority  
3. Search & Filter
4. Status workflow

**Учир:** Quality > Quantity. 4-ийг сайн ойлгож хийсэн нь 5-ыг хагасчилснаас илүү.

## Session 3 — Architecture ойлгох (Day 2-3)

**Огноо:** 2026-05-02 ~ 2026-05-03
**Зорилго:** Layered architecture-ийг ойлгож тайлбарлаж сурах

### Сурсан зүйлс

1. **4 layer:** Routes → Controller → Service → Repository
2. **Чиглэл:** Layer бүр зөвхөн **доод** layer-ээ дуудна
3. **Зарчим:** Separation of Concerns
4. **Үр ашиг:** DB сольсон ч Controller-ыг засахгүй, тестлэхэд хялбар

### AI-тай ярилцлагаас гарсан жишээ

**Q:** "Зөвхөн 1 layer ашигласан бол яах вэ?"

**A:** Бүх юм нэг файлд орж ирнэ — асуудал гарвал хайхад хэцүү, тестлэхэд хэцүү, олуул хамт ажиллахад мөргөлдөнө.

## Session 4 — Stack харьцуулалтын нарийн (Day 3)

**Огноо:** 2026-05-03
**Зорилго:** Express 4 vs 5, async support зэрэг ойлголтыг тодруулах

### Сурсан зүйлс

1. **Express 4 vs 5:** Express 5 шинэхэн (2024 stable) — middleware compatibility шалгагдаагүй. Express 4 нь тогтвортой, ecosystem бүрэн дэмжинэ. Тиймээс Express 4.19+ сонгов.

2. **Async support төрөлх:** Програм олон ажлыг зэрэг хийх боломжтой гэсэн утгатай. Node.js болон Python хоёр хоёулаа async-ыг сайн дэмждэг.

3. **Go-г хасах гол шалтгаан:** Шинэ хэл — 2 долоо хоногт хангалтгүй. Бас training data бага учир AI hallucination илүү.
